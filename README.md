# Практическая работа 2

По коду есть комментарии с описанием тех или иных методов. По структуре (основное):
- `Main` - запуск
- `Handler` - основная логика (создание сообщений, вызов продусера/консьюмеров)
- `BsonMapper` - вспомогательная утилита для сериализации/десериализации
- `MessageProducerImpl` - продусер
- `SingleMessageConsumer` - консьюмер с вычиткой по 1 сообщению
- `BatchMessageConsumer` - консьюмер с вычиткой по 10+ сообщений

### При запуске локально
- Необходимо запустить `/docker/kafka/docker-compose.yaml`
- Использовать нужную команду для создания топика под Вашу ОС из topic.txt внутри контейнера kafka
- Поправить поля с подключением к брокерам в `MessageProducerImpl`, `SingleMessageConsumer`, `BatchMessageConsumer` (есть комменты, сейчас под `docker`)
- Запустить main

### При запуске через docker-compose
- Необходимо запустить `/docker/kafka/docker-compose.yaml`
- Использовать нужную команду для создания топика под Вашу ОС из topic.txt внутри контейнера kafka
- Запустить второй `docker-compose.yaml` из корня проекта
